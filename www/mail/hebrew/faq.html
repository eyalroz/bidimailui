שאלות שכיחות

התקנה והסרה

התקנתי אצלי איזו גירסה של ההרחבה hebmailpack של מופי. מה אני צריכ/ה לעשות לפני התקנת התוסף BiDi Mail UI?

זה תלוי באיזו גירסה של התוסף של מופי יש לך, שכן מופי הכן גם גירסה מיוחדת ל-Thunderbird. כדי לבדוק האם זוהי הגירסה המותקנת אצלך, גש/י ללוח ההרחבות של Thunderbird. אם את/ה רואה את ההרחבה של מופי (הווה אומר, רואה את המילה hebmailpack או משהו שמזכיר דוא"ל בעברית או BiDi) - הסיר/י אותה ככל הרחבה ל-Thunderbird - באמצעות כפתור ה-Uninstall.

אם ההרחבה לא מופיע בלוח, או שאת/ה משתמש/ת במוזילה ולא ב-Thunderbird, תיאלץ/י להסיר את ההתקנה הישנה ידנית: פעל/י לפי ההוראות כאן, תוך חיפוש השם hebmailpack במקום bidimailpack.

שימ/י לב: אם ההרחבה של מופי מותקנת ב-Thunderbird ואינה מופיעה בלוח ההרחבות, יש להסיר גם אותה ידנית - לכן במקרה זה הוראות ההסרה עבור מוזילה תקפות גם עבור Thunderbird!

עתה תוכל/י להתקין את ההרחבה החדשה.

זיהוי כיוון

מישהו שלח לי איזו הודעה, אבל התוסף לא קבע נכון את הכיוון שלה. למה?

למצב כזה יש מספר סיבות אפשריות:

  • מוזילה טעה בזיהוי הקידוד של ההודעה, לכן ההרחבה אינה מודעת שיש בה באמת תוים בשפה הנכתבת מימין-לשמאל.
  • ההודעה שקיבלת כוללת מלל משני הסוגים - כזה הנכתב מימין-לשמאל וכזה הנכתב משמאל-לימין; מה שקורה הוא שאלגוריתם קביעת הכיוון פשוט עובד אחרת ממה שהאינטואיציה שלך מורה לגבי ההודעה הזו.
  • זוהי הודעת HTML ולא הודעת מלל פשוט.
  • קבעת את ההעדפה mailnews.message_display.autodetect_direction לערך false ובכך בחרת שלא יבוצע זיהוי כיוון אוטומטי.

התנהגות המקלדת

למה אי אפשר להשתמש ב-Ctrl+LeftShift ו-Ctrl+RightShift כמקשי-קיצור לשליטה בכיוון, כמו בחלונות ובמערכי מקשים נפוצים לעברית ב-X?

אנו מקווים לאפשר זאת בסופו-של-דבר, אך כרגע נבצר הדבר מאיתנו, בשל בעיות בטיפול בקלט מן המקלדת במוזילה.

מענה להודעות

כשאני עונה להודעה הכתובה מימין לשמאל, קו ציון הציטוט ממשיך להופיע משמאל לתוכן ההודעה המקורית, ולא עובר לצד ימין. האם אפשר לגרום לו להתנהג אחרת?

בעייה זו נפתרה בגירסה 0.5.1 של התוסף.

אם תרצה/י, הנה כמה חוקים להוספה ל-userContent.css אשר ישנו את צבע מלל הציטוט בהתאם לעומק הציטוט:

blockquote[type="cite"] { color: #660066 !important; } blockquote[type="cite"] blockquote { color: #007777 !important; } blockquote[type="cite"] blockquote blockquote { color: #990000 !important; } blockquote[type="cite"] blockquote blockquote blockquote { color: #000099 !important; } blockquote[type="cite"] blockquote blockquote blockquote blockquote { color: #bb6600 !important; }

כמו-כן זכור כי ניתן למנוע את הופעת קו הציטוט לחלוטין ולראות את סימני הציטוט הרגילים (סימני '>'), באמצעות קביעת ערך false להעדפה mail.quoted_graphical; כדי לעשות זאת ניתן להשתמש ב-URL about:config במוזילה, או להוסיף את השורה הבאה לקובץ user.js שלך:

pref("mail.quoted_graphical", false);

תשובה זו הועתקה בחלקה ממופי :-)

שונות

שמתי לב שחלק מקוד-המקור שבהרחבה BiDi Browser UI מצא את דרכו לבסיס-הקוד של מוזילה. האם נוכל לצפות שכך יקרה גם בהרחבה הזו? מתי צפוי הדבר להתרחש?

איזו דקות-אבחנה מרשימה... ובכן, ייתכן שחלק מן הקוד של ההרחבה יאומץ בשל כלשהוא בעתיד. הנושא פתוח לדיון בערוץ ה-IRC #bidi@moznet.

מדוע משנה ההרחבה את התנהגות מקש ה-Enter? אני רוצה את ההתנהגות ה-'Body Text'-ית הישנה והטובה חזרה...

תוכלו לחזור להתנהגות ברירת-המחדל של מקש ה-Enter באמצעות קביעת ההעדפה הבאה:

pref("mailnews.alternative_enter_behavior", false);

יש לי שאלה שלא מופיעה בדף הזה...
