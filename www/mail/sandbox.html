Installation & Removal

I have some version of Moofie's hebmailpack installed. What should I do before installing BiDi Mail UI?

That depends on whether you have the version Moofie made specifically for Thunderbird. To check whether you have that version installed, take a look at the Thunderbird extension panel. If you see Moofie's extension (the word 'Hebmailpack', or something mentioning Hebrew Mail or BiDi) - remove it as you would normally remove a Thunderbird extension - using the Uninstall button.

If the extension does not appear in the panel, or if you're using the Mozilla suite, you will have to manually remove the old extension: Follow the instructions here, using the name hebmailpack instead of bidimailpack. Note: Moofie's extension must also be manually removed from Thunderbird if it does not appear in the extension panel, so in this case the removal instructions apply to both Thunderbird and Mozilla suite!

Now install the new extension.

Automatic Direction Detection

I recieved this message, and the extension did not set its direction correctly. Why?

This could be due to one of the following:

  • Mozilla has mis-detected the message encoding, so the extension is not aware that there are actually any RTL-script characters in the message.
  • Your message contains both RTL and LTR text, and the detection algorithm simply works differently than your intuition for the message.
  • The message is in HTML rather than plain text.
  • You have disabled automatic message direction detection in the preferences dialog.

Keyboard Behavior

After installing BiDi Mail UI, As I compose new messages, I feel there's something weird about the behavior of the Enter key; what gives?

Without the extension, pressing Enter in the composer window inserts a line break (i.e. the <br> HTML tag). The extension indeed changes the behavior, and now pressing Enter 'inserts a paragraph break' (e.g. inserts a paragraph-ending </p>, followed by a new paragraph-beginning <p>). This behavior is similar to that of word processors such as OpenOffice Writer and Microsoft Office. Following this line of similarity, with the extension installed you can insert a line break by pressing Shift+Enter.

Why does direction switching not use the keyboard shortcuts Ctrl+LeftShift and Ctrl+RightShift, like in Microsoft Windows and popular Hebrew keyboard layouts for X?

We are hoping to eventually do so, but we are not yet able to, due to problems in keyboard input handling in Mozilla.
